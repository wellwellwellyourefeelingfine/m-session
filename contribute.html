Contribute

Healing the world, one person at a time.

M-session is a community project — free, open source, and built to help one person have a better, safer experience. Code, test, give feedback, or spread the word. Every contribution extends that reach to someone, somewhere, who needs it.

Contribute code

m-session is open source under AGPL-3.0. The entire codebase is on GitHub.

Tech stack

  • React 19 + Vite 7
  • Zustand for state management
  • Tailwind CSS 4
  • Vanilla JS for landing pages
  • ElevenLabs TTS for meditation audio

Getting started

  1. Clone the repository from GitHub
  2. Run nvm use (Node.js 22 LTS required)
  3. Run npm install
  4. Run npm run dev to start the dev server

The architecture is documented in the README. Look for issues labeled “good first issue” for a starting point, or open an issue to discuss a feature idea before building.

Help test

Testing a session companion app requires real-world use. Even quick checks are valuable. Here’s how you can help:

  • Use the app and report bugs via GitHub Issues
  • Test on different devices and browsers (especially iOS Safari)
  • Report audio playback issues — timing, volume, gaps
  • Test PWA installation and offline functionality
  • Check accessibility — screen readers, keyboard navigation, color contrast
  • Test the full session flow from intake through follow-up

Share feedback

Your experience using m-session is the most valuable input. If you’ve used the app, your perspective matters — whether positive, critical, or somewhere in between.

  • What worked well? What felt supportive?
  • What felt off, confusing, or missing?
  • Which modules resonated, and which didn’t?
  • Did the pacing feel right for your experience?
  • Was the design easy to use in an altered state?

Use the Give Feedback button in the app’s menu, or fill out our feedback form directly. You can also open a GitHub Issue for bug reports or feature requests.

Support the project

m-session is free and will always be free. But development, hosting, and audio production have real costs. Financial contributions directly support:

  • Audio recording and production (TTS generation for guided meditations)
  • Hosting and domain costs
  • Development time and tooling
  • Research and content development

Financial support options are coming soon. In the meantime, the most valuable contribution is your time — through code, testing, or feedback.

Tell someone

The most impactful contribution doesn’t require any technical skill. If m-session helped you, tell someone who might benefit.

Word of mouth is how tools like this reach the people who need them. Share with someone who’s exploring intentional use, someone in harm reduction communities, or someone who might appreciate having structure for their experience.

No social media pressure. Just genuine sharing with people who might benefit.
